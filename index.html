Project Name: Multi-Tier Website Deployment on AWS

Tools Used:

  • AWS EC2
  • AWS VPC
  • AWS RDS
  • AWS CloudFormation
  • AWS Auto Scaling
  • AWS Elastic LB

Description: I designed and implemented a scalable, highly available multi-tier website architecture for Company ABC using AWS services. The solution migrated their existing MySQL database and PHP website to a cloud-native infrastructure. I set up Amazon RDS for MySQL to ensure database reliability and easy management. For the web tier, I deployed EC2 instances running the PHP application within an Auto Scaling group to automatically adjust capacity based on demand. An Elastic Load Balancer was implemented to distribute traffic across multiple Availability Zones, enhancing fault tolerance. I configured VPC with public and private subnets to isolate the database and application layers, improving security. Security groups were meticulously set up to control inbound and outbound traffic. To ensure consistency and ease of deployment, I created AWS CloudFormation templates for infrastructure as code. Additionally, I implemented AWS Systems Manager for streamlined patch management and configuration of EC2 instances. This architecture not only provided the high availability and auto-scaling capabilities requested by Company ABC but also significantly improved their application\'s performance, security, and scalability in the cloud environment.

Project Name: Secure Patient Records Distribution System

Tools Used:

  • Amazon EC2
  • Amazon VPC
  • Amazon SNS
  • AWS CloudFormation
  • AWS SDK
  • Amazon RDS
  • AWS IAM
  • Amazon CloudWatch

Description: led the development of a secure and private platform for a healthcare provider to distribute patient records and send notifications. The system addresses the critical need for privacy and security in handling sensitive medical information. Using AWS CloudFormation, I automated the creation of a custom VPC to isolate our infrastructure. Within this VPC, I deployed EC2 instances to host our application server, built with Node.js and Express.js. The core of our solution leverages Amazon SNS for private message publishing, ensuring that patient reports are securely transmitted only to authorized recipients. I implemented VPC endpoints to establish a private connection between our VPC and Amazon SNS, enhancing security by keeping all traffic within the AWS network. For data persistence, I set up Amazon RDS with encryption at rest to store patient information securely. The application server uses the AWS SDK to interact with SNS and other AWS services. To manage access control, I configured IAM roles and policies, ensuring least privilege principles. For monitoring and alerting, I integrated Amazon CloudWatch to track system health and security events. The mobile app component allows patients to securely access their reports and receive push notifications. This solution not only meets HIPAA compliance requirements but also provides a scalable and maintainable platform for secure healthcare communication.
